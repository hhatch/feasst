    "# Tutorial\n",
    "\n",
    "## Installation and Interface\n",
    "\n",
    "Installation is described in the README of the base directory of FEASST.\n",
    "Or visit the website at https://pages.nist.gov/feasst\n",
    "\n",
    "### Python or C++?\n",
    "\n",
    "Although this tutorial will focus upon the python interface, FEASST may also be utilized as a C++ library.\n",
    "For a C++ example, see the file `tutorial/tutorial.cpp` which is virtually identical to the Python tutorial.\n",
    "Thus Python tutorials are sufficient for learning the C++ library.\n",
    "While the majority of FEASST users prefer the Python interface, FEASST is written almost entirely in C++ for speed.\n",
    "Thus, both interfaces will be supported in the long term.\n",
    "\n",
    "Some HPC clusters may not have the required Python libraries.\n",
    "If that is the case, do not hesitate to give the C++ interface a try, even if you have never written C++ before.\n",
    "For example, the only minor differences between `tutorial/tutorial.cpp` and `tutorial/tutorial.py` are the argument parsing syntax, semi-colons at the end of every line, and compiling any binary with `int main()`.\n",
    "\n",

    "The second step is to add a [Configuration](../plugin/configuration/README.rst).\n",

   ]
  },
  {
   "cell_type": "markdown",
   "metadata": {},
   "source": [
    "FEASST arguments are input as a dictionary of strings with limited type checking.\n",
    "Thus, care must be taken to input strings which follow the documentation.\n",
    "\n",
    "If there is a typo in the arguments, this may result in an exception which will print to the terminal and crash the notebook kernel. You can test this now by changing the `cubic_box_length` argument to a non-existent argument such as `cubic_box_len`.\n",
    "\n",
    "Next, initializing the [Potential](../plugin/system/doc/Potential.rst) proceeds as follows:"

    "monte_carlo.add(feasst.MakePotential(feasst.MakeLennardJones()))\n",

   ]
  },
  {
   "cell_type": "markdown",
   "metadata": {},
   "source": [
    "In this example, we introduce both the pair-wise [Lennard-Jones (LJ) model](../plugin/system/doc/LennardJones.rst), and also [long-range corrections](../plugin/system/doc/LongRangeCorrections.rst), which approximately account for the cut off of the LJ potential by assuming a pair-wise radial distance distribution function of unity.\n",
    "A FEASST convention is to use a helper function which appends the word `Make` onto the class name when creating pointers to FEASST derived class objects.\n",
    "This serves two purposes involving C++11 smart pointers and brace enclosed initializer lists.\n",
    "\n",
    "Initialize [ThermoParams](../plugin/system/doc/ThermoParams.rst), such as temperature, and the acceptance [Criteria](../plugin/monte_carlo/doc/Criteria.rst).\n",
    "Note that the initial configuration will be generated with grand canonical particle additions, so a large chemical potential is included, but will not contribute to canonical ensemble production simulations."

  },

    "This parameter may be adjusted to obtain a desired acceptance ratio, `tunable_target_acceptance`, with the help of [Tune](../plugin/steppers/doc/Tune.rst).\n"

    "monte_carlo.add(feasst.MakeTune(feasst.args({\"steps_per\" : steps_per})))"

  },

    "\n",
    "Additional [Analyze](../plugin/monte_carlo/doc/Analyze.rst) or [Modify](../plugin/monte_carlo/doc/Modify.rst) may be added at any time to perform some task contingent upon the number of attempted trials."
